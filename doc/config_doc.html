Config file structure

The json consists of a two objects 'appServerInfo' - parameters of internal server and 'externalServerInfo' - parameters of external server. Important: Fields are clickable, just click on them!

{
★

"appServerInfo":

Information about the application server.
"type":
object
★"serverIp":

IP address of the application server.
"type":
string
"example":
127.0.0.1
★"serverPort":

Port number of the application server.
"type":
string
"example":
8080
★"amountOfWorkers":

Number of worker on server, which will process the requests.
"type":
integer
"minimum":
1
"example":
8
★"queueSize":

Maximum size of the queue. All request will be placed there. If lenght of it will be exceeded then user will get an error.
"type":
integer
"minimum":
1
"example":
10
★"ttlOfItemsInCache":

Time-to-live (TTL) of items in the cache (in seconds).
"type":
integer
"minimum":
1
"example":
20000
★"logLevel":

Log level for the application.
"type":
string
"enum":
"0":
DEBUG
"1":
INFO
"2":
WARNING
"3":
ERROR
"example":
DEBUG
★

"externalServerInfo":

Information about the external server.
"type":
object
★"employeeUrlPath":

URL path for employee data on the external server, which must provide person's id by email given.
"type":
string
"format":
uri
"example":
https://127.0.0.1:8081/Portal/springApi/api/employees
★"absenceUrlPath":

URL path for absence data on the external server, which must provide person's reason id of absence by id of the person.
"type":
string
"format":
uri
"example":
https://127.0.0.1:8081/Portal/springApi/api/absences
★"requestTimeout":

Timeout for external server requests (in milliseconds).
"type":
integer
"minimum":
50
"example":
2000
★"login":

Login credentials for external server authentication.
"type":
string
"example":
somelogin
★"password":

Password credentials for external server authentication.
"type":
string
"example":
password
}
